# CS340
Throughout this course, I learned that writing maintainable, readable, and adaptable programs starts with organization and separation of responsibility. In Project One, I created a CRUD Python module that handled all database interactions independently from the dashboard interface. By separating the database logic from the user interface, I made the system easier to manage and debug. Instead of embedding MongoDB queries directly inside the Dash app, the dashboard simply called methods from the AnimalShelter class. This improved readability and made the code easier to maintain. If changes were needed in the database structure or authentication, I only had to modify the CRUD module rather than the entire application. In the future, I could reuse this CRUD module for other dashboards, reporting tools, or even integrate it into a larger web application. The modular design makes it adaptable and scalable.

When approaching problems as a computer scientist, I focus on breaking the problem into smaller components and identifying the relationship between them. For the Grazioso Salvare dashboard, I first considered the database requirements: what data was needed, how it should be filtered, and how users would interact with it. I then designed the database connection before building the interface. This differed from some earlier assignments where the focus was more theoretical or isolated to small scripts. In this project, I had to think about full system interaction — database, backend logic, and front-end visualization working together. In the future, I would continue using structured planning techniques such as outlining requirements first, designing the data model carefully, and ensuring proper indexing for performance before building the user interface.

Computer scientists design systems that solve real-world problems through structured logic and efficient data management. In this project, the dashboard allows Grazioso Salvare to quickly filter dogs by rescue type, view breed distributions, and analyze geographic intake data. This reduces manual searching and improves decision-making efficiency. By organizing data properly and presenting it visually, the system supports faster and more informed rescue placement decisions. Projects like this demonstrate how software development directly impacts organizational effectiveness, accuracy, and productivity.
